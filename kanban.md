### ブロックスキップ関係のフォーカス修正

+ 親：[ブロックスキップの問題解消](#ブロックスキップの問題解消)
+ 作業範囲：CSS/WordPress

`id="anchor-*" tabindex="-1"` の箇所において、見栄えが悪くなる問題が発生している。  
これはレイアウト要素をフォーカス可能にした関係で、クリックするとレイアウト部にフォーカスが当たるようになったためである。

必要となる結果は、下記のとおりである。

+ クリックしてもレイアウトにフォーカスのアウトラインが発生しないこと
+ tabキーによる操作にはフォーカス時にアウトラインがあたること
+ ブロックスキップ時においては、アウトラインがあたった方が望ましい

これを基に、下記の検証と修正を行う。

1. 疑似クラス `focus-visible` を使いユーザーの入力方法に応じた使い分けを行う
2. `id="anchor-*"` で前方一致の要素に対しスタイルを追加

作業における追記

+ 1番の `focus-visible` の手法で必要な結果を達成できた
+ また、「ページトップに戻る」ボタン押下時にヘッダーにフォーカスがあたる問題も解消できた
+ 上記はTabキーを使った場合はヘッダーにフォーカスエフェクトが生じているので、より望ましくなっている

注意：比較的新しい機能のため、古いブラウザでは機能しない可能性がある。
そのため、検証し問題があるようなら再度修正を行う必要がある。
[MDNによれば、2022年3月移行のバージョンに対応しているらしい](https://developer.mozilla.org/ja/docs/Web/CSS/:focus-visible)

+ 親：[開発ツールの有効無効の切り替え](#開発ツールの有効無効の切り替え)
+ 作業範囲：HTML/CSS/Javascript/WordPress

現在の状況は、開発ツール有効時にカラーモードの切り替えにOS、ブラウザの設定を反映できていない。
開発ツールがある状態だと必ず `html` タグに `is_darkMode/is_lightMode` のクラスが存在し、CSSはそちらの設定を優先する。

これを改善するために、フォントサイズ変更機能と同様「カラーモードの手動切り替え」用のチェックボックスを新たに設ける。
チェックされてない状況下では `html` タグにクラスを付与する挙動を行わないようにする。  
（なおフォントサイズの機能同様、この設定も含めて `sessionStorage` で同セッション中は設定内容を維持できるようにする）

### 要素セレクタに対するスタイル剥ぎを変更

+ 作業範囲：CSS

要素セレクタに対するスタイルを剥ぐ設定を削除し、影響を受ける箇所に個別に設定していく。

現在 `summary`, `button`, `select` の3つの要素について、`style.css` でスタイルを剥ぐ挙動を取っている。
それによる影響を受けているクラス（ID）は、下記のようになっている。

+ `summary` タグ
  + bl_accordion
+ `select` タグ
  + bl_searchForm_inputSelect（`div` タグ内）
+ `button` タグ
  + bl_header_btnIcon（`a` タグを使う場合もある）
  + bl_menuHeader_btnIcon
  + bl_searchForm_btn
  + el_btn（`a` タグを使う場合もある）
  + js_pageTop（ID）

これらの要素は既に確定しているものなので個別に規定することができ、要素セレクタを使う必要性は薄れている。

また、ノーマライズCSSを使用する趣旨として
「基本はデフォルトのスタイルは維持してブロックコンテンツ側のスタイルに影響を与えないようにしよう（`wp_blockContent` での最低限のスタイルは除いて）」
という理由が存在する。その趣旨に照らし合わせると、要素セレクタのスタイルを剥ぐ設定は望ましくないと判断した。
